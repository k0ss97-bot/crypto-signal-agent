# Спецификация MVP: крипто-аналитический агент OpenAI

Дата версии: 2026-07-05

Важно: это не финансовая рекомендация. Агент не должен сам покупать или продавать монеты на первом этапе. Его задача — находить события, проверять доступность монеты на Bybit и Binance, оценивать риск и отправлять понятный сигнал на русском языке.

## Цель

Построить агента, который:

- отслеживает события по криптомонетам;
- проверяет источник;
- проверяет, есть ли торговая пара на Bybit Spot;
- по умолчанию дополнительно проверяет Binance Spot;
- оценивает цену, объем, spread и ликвидность;
- выдает сигнал на русском языке;
- сохраняет журнал сигналов для последующей проверки результата.

Названия сервисов, сайтов, бирж и монет не переводятся: Bybit, Binance, OpenAI, Telegram, `ABCUSDT`.

## Основное торговое правило

```text
PRIMARY_EXCHANGE=bybit
REQUIRED_EXCHANGES=bybit,binance
REQUIRE_ALL_EXCHANGES=true
SCAN_ASSETS=BTC,ETH,SOL,BNB,XRP
```

Смысл:

- если пары нет на Bybit Spot, сигнал блокируется всегда;
- если пары нет на Binance Spot, при `REQUIRE_ALL_EXCHANGES=true` сигнал блокируется;
- если поставить `REQUIRE_ALL_EXCHANGES=false`, отсутствие Binance станет предупреждением, но отсутствие Bybit все равно будет блокировкой.

## Архитектура

```text
Источники данных
  -> сборщик событий
  -> проверка источника
  -> проверка Bybit и Binance
  -> сбор рыночных метрик
  -> скоринг
  -> риск-фильтр
  -> анализ OpenAI
  -> Telegram / журнал SQLite
```

## Источники данных

| Источник | Зачем нужен |
|---|---|
| Bybit Spot API | основная проверка торговой пары, цены, объема и spread |
| Binance Spot API | подтверждение листинга и дополнительной ликвидности |
| Bybit / Binance announcements | листинги, делистинги и важные объявления |
| CoinGecko API | рыночная капитализация, объемы и справочные данные |
| Официальные сайты проектов | подтверждение новостей |

## Типы событий

| Событие | Как трактовать |
|---|---|
| спотовый листинг | возможный импульс, но без покупки “в догонку” |
| фьючерсный листинг | умеренно позитивно, риск перегрева выше |
| делистинг | негативный сигнал |
| взлом или эксплойт | сильный негатив |
| крупная интеграция | позитивно только при официальном источнике |
| разблокировка токенов | риск давления продаж |
| аномальный объем | сигнал только после проверки причины |

## Скоринг

| Фактор | Вес |
|---|---:|
| официальный источник | +25 |
| спотовый листинг на крупной бирже | +25 |
| фьючерсный листинг | +10 |
| цена уже выросла на 40-80% | -20 |
| объем выше среднего более чем в 3 раза | +15 |
| ликвидность достаточная | +10 |
| нет одной из обязательных бирж | -30 |
| взлом, эксплойт или делистинг | -40 |
| источник не подтвержден | -25 |

## Риск-фильтр

Жесткие правила MVP:

```text
LIVE_TRADING_ENABLED=false
NO_SIGNAL_IF_SOURCE_UNVERIFIED=true
MAX_SPREAD_PCT=0.35
MIN_VOLUME_RATIO_VS_7D=1.0
MIN_LIQUIDITY_OK=true
```

Риск-фильтр блокирует сигнал, если:

- источник не подтвержден;
- пары нет на Bybit Spot;
- нет обязательной подтверждающей биржи при строгом режиме;
- spread выше лимита;
- ликвидность недостаточная;
- случайно включен флаг live trading.

## Правило языка

Все сообщения для человека должны быть на русском:

- Telegram-уведомления;
- объяснение OpenAI;
- причины блокировок;
- предупреждения;
- CLI-вывод;
- README и пользовательская документация.

Не переводятся:

- Bybit, Binance, OpenAI, Telegram;
- URL сайтов;
- тикеры и торговые пары;
- технические имена переменных окружения и команд.

## Следующие шаги реализации

1. Подключить реальные рыночные метрики Bybit.
2. Использовать команду `scan` для самостоятельной проверки списка монет.
3. Добавить сбор объявлений Bybit и Binance.
4. Добавить Telegram-уведомления по расписанию.
5. Сохранять результат цены через 5 минут, 1 час, 4 часа и 24 часа.
6. После накопления статистики добавить paper trading.
7. Только после проверки статистики обсуждать semi-auto execution.

## Команда самостоятельной проверки

```bash
python -m crypto_signal_agent.cli scan
```

Проверить конкретные монеты:

```bash
python -m crypto_signal_agent.cli scan --assets BTC,SOL,LINK
```

Проверить и отправить отчет в Telegram:

```bash
python -m crypto_signal_agent.cli scan --assets BTC,SOL,LINK --send-alert
```

## Мониторинг новых монет

Первый запуск:

```bash
python -m crypto_signal_agent.cli monitor-new --send-alert --send-empty
```

Этот запуск сохраняет текущий Bybit Spot как baseline и не считает старые пары новыми.

Постоянный мониторинг:

```bash
python -m crypto_signal_agent.cli monitor-new --loop --send-alert
```

Логика:

- бот загружает весь список Bybit Spot;
- оставляет только пары с котировкой `USDT`;
- сравнивает с локальной SQLite-базой известных пар;
- если появилась новая пара, проверяет Binance;
- формирует русский сигнал;
- отправляет сигнал в Telegram;
- если новых монет нет, Telegram молчит, чтобы не спамить.
